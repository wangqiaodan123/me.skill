# 自我.skill — 产品需求文档（PRD）

## 产品定位

自我.skill 是一个运行在 Claude Code 上的 meta-skill。用户通过对话式交互提供原材料（聊天记录 + 照片 + 手动描述），系统自动生成一个可独立运行的数字分身 Skill。

## 核心概念

### 两层架构

| 层 | 名称 | 职责 |
|----|------|------|
| Part A | Life Memory | 存储个人经历：人生时间线、重要事件、价值观、兴趣爱好 |
| Part B | Persona | 驱动对话行为：说话风格、情感模式、人际行为 |

两部分可以独立使用，也可以组合运行。

### 运行逻辑

```
别人发消息 → Part B 判断你会怎么回应 → Part A 补充人生经历 → 用你的方式输出
```

### 进化机制

```
追加原材料 → 增量分析 → merge 进现有 Skill
对话纠正 → 识别修正点 → 写入 Correction 层
版本管理 → 每次更新自动存档 → 支持回滚
```

## 用户旅程

```
用户触发 /create-me
  ↓
[Step 1] 代号/基本信息/自我画像
  - 代号/昵称
  - 基本信息（职业、年龄段、所在地等）
  - 自我画像（MBTI、性格标签、价值观、自我描述）
  ↓
[Step 2] 原材料导入（可跳过）
  - 微信聊天记录导出
  - QQ 聊天记录导出
  - 社交媒体截图
  - 照片
  - 直接粘贴/口述
  ↓
[Step 3] 自动分析
  - 线路 A：提取人生记忆 → Life Memory
  - 线路 B：提取性格行为 → Persona
  ↓
[Step 4] 生成预览，用户确认
  - 分别展示 Memory 摘要和 Persona 摘要
  - 用户可直接确认或修改
  ↓
[Step 5] 写入文件，立即可用
  - 生成 selves/{slug}/ 目录
  - 包含 SKILL.md（完整组合版）
  - 包含 memory.md 和 persona.md（独立部分）
  ↓
[持续] 进化模式
  - 追加新文件 → merge 进对应部分
  - 对话纠正 → patch 对应层
  - 版本自动存档
```

## 安全边界

1. **仅用于创建个人数字分身**
2. **不主动联系真人**
3. **不鼓励过度依赖数字分身**
4. **数据仅本地存储**
5. **Layer 0 硬规则**保证不说你绝不可能说的话

## 数据源支持矩阵

| 来源 | 格式 | 提取内容 | 优先级 |
|------|------|---------|--------|
| 微信聊天记录 | txt / html / json / csv exports | 完整对话、语气词、回复模式 | ⭐⭐⭐ |
| QQ 聊天记录 | txt/mht | 完整对话 | ⭐⭐⭐ |
| 照片 | JPEG/PNG + EXIF | 时间线、地点 | ⭐⭐ |
| 朋友圈/微博截图 | 图片 | 公开人设、兴趣 | ⭐⭐ |
| 口述/粘贴 | 纯文本 | 主观记忆 | ⭐ |

## 文件结构

```
selves/
  └── {slug}/
      ├── SKILL.md          # 完整组合版，可直接运行
      │                     # 触发词: /{slug}
      ├── memory.md         # Part A：人生记忆
      │                     # 触发词: /{slug}-memory
      ├── persona.md        # Part B：人物性格
      │                     # 触发词: /{slug}-persona
      ├── meta.json         # 元信息
      ├── versions/         # 历史版本存档
      └── memories/         # 原始材料存放
          ├── chats/
          ├── photos/
          └── social/
```
